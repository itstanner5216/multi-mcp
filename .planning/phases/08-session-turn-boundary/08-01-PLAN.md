# Plan 08-01: Session State & Turn Boundary Discipline

## Phase 7 Carry-Forward Corrections Absorbed by Phase 8

Phase 7 delivered a working BMXF scoring pipeline (1031 tests passing) but left four correctness gaps that directly affect session isolation, turn-boundary semantics, and faithful implementation of the source plan. These gaps are not optional cleanups. They are Phase 8 mandatory corrections because deferring them further would allow tests written against incorrect behavior to accumulate, making later correction more expensive.

This plan treats all four as first-class Phase 8 scope. Each appears in objectives, must_haves, model changes, files_modified, tests, and the invariant coverage matrix.

### CF-1 — Shared Fallback Session Identity

**Current defect:** `mcp_proxy.py _get_session_id()` (line 356) falls back to `"default"` when `self._server_session is None`. Any retrieval-path call made before a session is active, or during test setup without a real session, silently runs under a shared identity. This violates session isolation (source plan line 325: *"SessionRoutingState is never shared across sessions"*).

**Required correction:** `_get_session_id()` must raise `RuntimeError` when `_server_session is None`. The proxy must not call any retrieval pipeline method without a real session. Per-session identity must be managed via `_session_ids: dict[int, str]` mapping `id(session_obj)` to a `uuid4` hex string, cleaned up on session teardown.

### CF-2 — Router Describe / Proxy Signal Conflation

**Current defect:** `pipeline.py` line 667 sets `session_router_proxies = list(self._session_router_describes.get(session_id, []))` — it populates the `router_proxies` field of `RankingEvent` from the describes dict, not a proxies dict. There is no separate `_session_router_proxies` dict. `router_proxies` and `router_describes` in `RankingEvent` are fed from the same source, making them identical. The `recent_router_proxies` field in `SessionRoutingState` is still `list[str]` and nothing writes actual proxy calls to it.

**Required correction:** `_session_router_describes` tracks describe-only routing calls (tool metadata requests). A new `_session_router_proxies` dict tracks actual proxied tool invocations via `request_tool`. These must be maintained separately and must feed `RankingEvent.router_describes` and `RankingEvent.router_proxies` from their respective sources. `state.recent_router_proxies` must be populated only from actual proxy calls.

### CF-3 — Session-Accumulated History Used as Turn-Boundary State

**Current defect:** `pipeline.py` line 644 sets `used_this_turn = set(self._session_tool_history.get(session_id, []))` — it reads the entire unbounded session history to determine which tools were "used this turn" for demotion protection. A tool used 20 turns ago receives demotion protection every turn thereafter. This is incorrect. Demotion protection must apply only to the just-finished turn.

**Required correction:** Phase 8 introduces explicit turn-scoped buckets: `_current_turn_used: dict[str, set[str]]` (written during a turn by `on_tool_called`) and `_just_finished_turn_used: dict[str, set[str]]` (snapshot rolled at turn boundary entry). Demotion protection reads `_just_finished_turn_used`, not `_session_tool_history`. `_session_tool_history` remains for conversation-context retrieval query construction only.

### CF-4 — Outdated SessionRoutingState Shape

**Current defect:** `models.py` line 132 declares `recent_router_proxies: list[str]`. This flat list loses per-turn information and cannot represent the source plan's "used via router 2/3 last turns" promotion criterion (source plan line 304). The pipeline does not instantiate `SessionRoutingState` per session and does not write to it. The `_routing_states`, `_in_turn`, `_pending_rebuild`, and `_turn_snapshot_version` dicts from the source plan do not exist yet.

**Required correction:** `recent_router_proxies` must be changed to `dict[str, list[int]]` (tool_key → sorted unique turn numbers, pruned to last 3). This is a data-model migration. All code paths that read or write session routing state must use the `_routing_states` dict. The `_in_turn`, `_pending_rebuild`, and `_turn_snapshot_version` dicts must be introduced and their write/read/clear paths must be explicit.

## Objective

Enforce session isolation with real per-session UUIDs and no shared fallback, move all active-set mutations to turn boundaries, wire true turn-scoped usage buckets for promote/demote decisions, split router-describe and router-proxy accounting into separate signals, upgrade `SessionRoutingState` to the shape required by the source plan, instantiate it per session, wire catalog snapshot pinning, and assemble the returned tool list from the post-boundary active set — not raw scored candidates.

After this plan:
- `_get_session_id()` raises instead of falling back — no retrieval code runs under a shared session
- `_session_router_describes` and `_session_router_proxies` are distinct dicts with one write path each; `on_tool_called(is_router_proxy=True)` is the single write path for proxy accounting
- Demotion protection reads `_just_finished_turn_used` (last-turn bucket), not `_session_tool_history` (unbounded history)
- `SessionRoutingState.recent_router_proxies` is `dict[str, list[int]]` and is written only by `on_tool_called(is_router_proxy=True)`
- `_routing_states`, `_in_turn`, `_pending_rebuild`, `_turn_snapshot_version` all exist and are maintained per session
- The tool list returned by `get_tools_for_list()` is built from the post-boundary active set from `SessionStateManager`, not from raw `scored_tools[:k]`
- Mid-turn stability invariant holds; session state never leaks

## Authority Model: SessionStateManager and SessionRoutingState

**`SessionStateManager` (session.py)** is the source of truth for live active membership mutations. It is the only component that adds or removes tools from the active set via `promote()` and `demote()`. No code path may change the live active set through `SessionRoutingState.active_tool_ids` directly.

**`SessionRoutingState` (models.py)** is the authoritative per-session routing/ranking metadata container for:
- `turn_number` — canonical turn counter
- `catalog_version` — snapshot version pinned at turn start
- `alpha`, `env_confidence`, `conv_confidence`, `fallback_tier`, `active_k` — per-turn scoring signals
- `recent_router_proxies` — turn-indexed router proxy history for the 2-of-3 promotion criterion
- `consecutive_low_rank` — per-tool demotion hysteresis counter
- `active_tool_ids` — post-boundary mirror of `SessionStateManager`'s active set for this session
- `router_enum_tool_ids` — post-boundary router enumeration set for this session

**Synchronization contract at turn boundary:**
1. `session_manager.promote(session_id, promote_candidates)` — mutates live membership
2. `session_manager.demote(session_id, demote_candidates, just_finished)` — mutates live membership
3. `state.active_tool_ids = list(session_manager.get_active_tools(session_id))` — mirrors post-boundary state
4. `state.router_enum_tool_ids` is set to all registry keys not in `state.active_tool_ids`
5. The returned tool list is assembled from `state.active_tool_ids`

**No split-brain:** `state.active_tool_ids` is always set from `session_manager.get_active_tools()` in the same boundary pass where promote/demote are called. There is no path where the two can diverge within a single turn boundary evaluation.

**SessionStateManager API note:** The current `session.py` API (`promote()`, `demote()`, `get_active_tools()`, `cleanup_session()`) is sufficient for Phase 8. No new methods are required. If a future phase needs to set the full active set without going through promote/demote (e.g., to reset to a catalog snapshot), `session.py` would need a `set_active_tools()` method. That is out of scope for Phase 8.

## Changes

### 1. Wire real session IDs with no fallback (CF-1 / F-02)

**File:** `src/multimcp/mcp_proxy.py`

**Current defect (line 354-356):**
```python
if self._server_session is not None:
    return str(id(self._server_session))
return "default"  # WRONG — shared fallback
```
Any retrieval-path call made when `_server_session is None` silently runs under `"default"`. This violates source plan line 325: *"SessionRoutingState is never shared across sessions."*

**Source plan (lines 324-325):** Per-session isolation is a core invariant. `SessionRoutingState` must be session-scoped.

**Required implementation — raises instead of falling back:**
```python
self._session_ids: dict[int, str] = {}  # id(session_obj) -> uuid4 hex

def _get_session_id(self) -> str:
    """Derive a stable session ID from the current MCP server session.

    Raises RuntimeError if called with no active session. The proxy
    must never call retrieval pipeline methods without a real session.
    """
    if self._server_session is None:
        raise RuntimeError(
            "No active MCP session; retrieval pipeline must not be called without a session"
        )
    obj_id = id(self._server_session)
    if obj_id not in self._session_ids:
        self._session_ids[obj_id] = uuid.uuid4().hex
    return self._session_ids[obj_id]
```

**Why `id(self._server_session)` + UUID dict, not raw `str(id(...))` directly:** Raw `id()` values are reused by the runtime after GC. Mapping them to stable UUIDs ensures the session_id remains stable for the lifetime of the session object and does not collide with a future session that happens to get the same memory address.

**Why raising is correct, not a fallback:** The only legitimate call context for retrieval methods is inside an active session. If the retrieval path is reachable without a session, that is a programming error that must be caught at the call site, not silently tolerated.

**Lifecycle invariant:** One session_id corresponds to exactly one MCP client connection. In SSE mode, different `_server_session` objects produce different IDs. In STDIO mode, one object produces one stable ID. Teardown removes the mapping and all pipeline state.

**Changes to mcp_proxy.py:**
- Add `import uuid` if not already present
- Add `self._session_ids: dict[int, str] = {}` to `__init__`
- Replace `_get_session_id()` body: remove `return "default"` branch; raise `RuntimeError` when `_server_session is None`; add `_session_ids` lookup/create logic
- Add `self._last_tool_list_hash: dict[str, str] = {}` to `__init__`
- Remove all remaining TODO comments acknowledging hardcoded `"default"`
- Session teardown (in `run()` after message loop exits): call `_get_session_id()` before `self._server_session = None`, store result, then call `pipeline.cleanup_session(session_id)`, `self._session_ids.pop(id(old_session_obj), None)`, `self._last_tool_list_hash.pop(session_id, None)`

### 2. Turn-boundary guard for tools/list_changed (F-07)

**File:** `src/multimcp/mcp_proxy.py`

**Current (lines 474-481):** After `on_tool_called()` returns `disclosed=True`, immediately calls `await self._send_tools_list_changed()`. This fires mid-turn — the model receives a tools/list_changed notification during its own tool-call sequence.

**Source plan (line 323):** Core Invariant: "Mid-turn stability: Active set and router enum do not change during a model turn."
**Source plan (lines 303-306):** "Promote/demote with hysteresis at turn boundary" — not during a turn.

**The turn boundary contract (one clean model, no alternatives):**

A "model turn" spans from when the model receives tools/list through all subsequent tool calls until the next tools/list request. The mid-turn stability invariant (source plan line 323) means: **no active-set mutation, no routing-enum mutation, no catalog rebuild, and no tools/list_changed notification during this entire span.**

**Implementation — `_in_turn` flag on the pipeline, scoped to the full turn:**

Section 7 is the single authoritative sequence. In summary:
- `get_tools_for_list()` entry = turn boundary: closes the previous turn (clears `_in_turn`, pops `used_last_turn`), then scores, promotes/demotes, and at exit sets `_in_turn[session_id] = True` for the new turn.
- `on_tool_called()` is always mid-turn (`_in_turn` is True): records usage only, never mutates active set.

The key invariant: `_in_turn[session_id]` is set to `True` at the END of `get_tools_for_list()` (step 9), not at the start. It is cleared to `False` at the ENTRY of the next `get_tools_for_list()` call (step 2).

**tools/list_changed:** Only emitted from `_list_tools()` when the tool list returned this call differs from the previous call's list. Never emitted from `_call_tool()`. The proxy tracks `_last_tool_list_hash: dict[str, str]` per session.

```python
# In _list_tools():
tools = await self.retrieval_pipeline.get_tools_for_list(session_id, conv_context)
new_hash = _hash_tool_list(tools)
if session_id in self._last_tool_list_hash and new_hash != self._last_tool_list_hash[session_id]:
    await self._send_tools_list_changed()
self._last_tool_list_hash[session_id] = new_hash
return types.ServerResult(tools=tools)
```

**rebuild_catalog() mid-turn guard:** When `_in_turn[any_session]` is True, `rebuild_catalog()` queues the rebuild into `_pending_rebuild` and defers execution to the next `get_tools_for_list()` call. This prevents `register_client()`/`unregister_client()` from mutating the catalog mid-turn.

**Changes to mcp_proxy.py `_call_tool()`:**
- Remove lines 474-481 entirely (the `on_tool_called` → `tools/list_changed` emission path)
- Replace with: `await self.retrieval_pipeline.on_tool_called(session_id, tool_name, arguments)` — no conditional, no notification

**Important: Turn counting semantics.** Turns are counted by `get_tools_for_list()` calls (one per model turn), NOT by `on_tool_called()` (multiple per turn). `_session_turns` must be incremented in `get_tools_for_list()`, and `SessionRoutingState.turn_number` is the canonical counter.

**Changes to pipeline.py `on_tool_called()`:**

```python
    # Write to turn-scoped bucket for demotion protection at next boundary (CF-3)
    self._current_turn_used.setdefault(session_id, set()).add(tool_name)
    # Write to session history for conv-context retrieval query construction only
    self._session_tool_history.setdefault(session_id, []).append(tool_name)
    # If this is a routing-tool proxy call, write all proxy accounting (CF-2)
    if is_router_proxy:
        # Flat session list for RankingEvent.router_proxies
        self._session_router_proxies.setdefault(session_id, []).append(tool_name)
        # Turn-indexed structure for 2-of-3-turn promotion criterion (CF-4)
        state = self._routing_states.get(session_id)
        if state is not None:
            turn = state.turn_number
            turns_list = state.recent_router_proxies.setdefault(tool_name, [])
            if not turns_list or turns_list[-1] != turn:  # unique turn entry
                turns_list.append(turn)
            cutoff = turn - 3
            state.recent_router_proxies[tool_name] = [t for t in turns_list if t >= cutoff]
    return False  # Never signal mid-turn changes
```

**`is_router_proxy=True` detection:** In `mcp_proxy.py _call_tool()`, when the routing tool dispatches a proxy call (the `__PROXY_CALL__:` sentinel path), call:
```python
await self.retrieval_pipeline.on_tool_called(session_id, tool_name, arguments, is_router_proxy=True)
```
This is the only call site that passes `is_router_proxy=True`. All other `on_tool_called()` calls use the default `is_router_proxy=False`. There is no `record_router_proxy()` method.

**Changes to mcp_proxy.py:**
- `_call_tool()`: Remove the `tools/list_changed` emission after `on_tool_called()` (lines 474-481). `on_tool_called()` now always returns False.
- `_list_tools()`: After getting tools from pipeline, compare with previous tool list. If changed, emit `tools/list_changed` after `get_tools_for_list()` returns, before sending the MCP response.

```python
    score_by_key = {s.tool_key: s.score for s in scored_tools}
```

### 6. Implement K-2 promotion criterion using turn-indexed router history (CF-4 / F-12)

**File:** `src/multimcp/retrieval/pipeline.py`

**Current:** Promote only via direct invocation (`on_tool_called` → `promote()`).

**Source plan (line 304):** "Promote: rank within K-2 OR used via router 2/3 last turns."

**The `recent_router_proxies` dict is the only representation for the 2-of-3 criterion.** No alternative accumulator exists. The flat `list[str]` is removed by the CF-4 migration in Section 3.

**Promote evaluation in `get_tools_for_list()` (at turn boundary, after scoring):**

```python
k_minus_2 = max(1, dynamic_k - 2)
active_key_set = set(state.active_tool_ids)
promote_candidates = []

# Criterion 1: rank within K-2
for scored_tool in scored_tools[:k_minus_2]:
    if scored_tool.tool_key not in active_key_set:
        promote_candidates.append(scored_tool.tool_key)

# Criterion 2: router-proxied in >=2 of last 3 turns
# Uses SessionRoutingState.recent_router_proxies: dict[str, list[int]]
# Written exclusively by on_tool_called(is_router_proxy=True) — see Section 6.5
current_turn = state.turn_number
for tool_key, turns in state.recent_router_proxies.items():
    recent = [t for t in turns if t >= current_turn - 3]
    if len(recent) >= 2 and tool_key not in active_key_set:
        promote_candidates.append(tool_key)

newly_promoted = self.session_manager.promote(session_id, promote_candidates)
state.active_tool_ids.extend(newly_promoted)
```

### 6.6. Replace session-accumulated history with turn-boundary buckets (CF-3)

**File:** `src/multimcp/retrieval/pipeline.py`

**Current defect (pipeline.py line 644):**
```python
used_this_turn = set(self._session_tool_history.get(session_id, []))
```
This reads the full session history — every tool ever called — as if it were the current turn's usage. The result: a tool used once, 20 turns ago, is protected from demotion for the entire session.

**Required turn-scoped state:**

Three distinct stores, each with an explicit write/roll/clear path:

| Store | Type | Written by | Rolled/cleared | Used for |
|-------|------|------------|----------------|----------|
| `_current_turn_used[sid]` | `set[str]` | `on_tool_called()` — each call | At turn boundary entry: pop → `_just_finished_turn_used[sid]` | Not read directly |
| `_just_finished_turn_used[sid]` | `set[str]` | Turn boundary roll | Overwritten each boundary | Demotion protection only |
| `_session_tool_history[sid]` | `list[str]` | `on_tool_called()` — each call | Never cleared (session-scoped) | Retrieval conv-context query only |

**Write path — `on_tool_called()`:**
```python
# Record in current-turn bucket (cleared at next boundary)
self._current_turn_used.setdefault(session_id, set()).add(tool_name)
# Record in session history (for conv-context retrieval, never used in promote/demote)
hist = self._session_tool_history.setdefault(session_id, [])
hist.append(tool_name)
```

**Roll path — `get_tools_for_list()` boundary entry (step 2 of state machine):**
```python
# Roll current turn → just-finished snapshot for demotion protection
self._just_finished_turn_used[session_id] = self._current_turn_used.pop(session_id, set())
```
This happens exactly once at the start of `get_tools_for_list()`, before scoring begins.

**Read path — demotion protection (Section 5):**
```python
just_finished = self._just_finished_turn_used.get(session_id, set())
demoted = self.session_manager.demote(session_id, demote_candidates, just_finished, max_per_turn=3)
```

**Invariant enforced:** `_session_tool_history` is never read by promote/demote logic. `_just_finished_turn_used` is never appended to after its roll. `_current_turn_used` is never read outside of `on_tool_called()` internals.

**Cleanup:** `cleanup_session()` must pop both `_current_turn_used[session_id]` and `_just_finished_turn_used[session_id]`.

### 7. Exact `get_tools_for_list()` state machine

`get_tools_for_list(self, session_id, conversation_context)` must execute in this exact order:

1. Derive `session_id` (already passed by proxy — the proxy raises before calling if no session)
2. **Turn boundary entry:** If `_in_turn[session_id]` is `True`, close the previous turn:
   - Set `_in_turn[session_id] = False`
   - Roll turn buckets: `self._just_finished_turn_used[session_id] = self._current_turn_used.pop(session_id, set())` (CF-3)
   - Otherwise (first call for this session): `self._just_finished_turn_used[session_id] = set()`
3. If `_pending_rebuild` is not None and no session is currently mid-turn (`not any(self._in_turn.values())`), execute the pending rebuild
4. Load or create `SessionRoutingState` from `_routing_states` for this session (CF-4); initialize `session_manager.get_or_create_session(session_id)` if needed
5. Increment `state.turn_number` by 1
6. Pin `state.catalog_version` from `self.retriever.get_snapshot_version()`; store in `_turn_snapshot_version[session_id]`
7. Score: run the fallback ladder (from Phase 7) to produce `scored_tools: list[ScoredTool]` ranked by score
8. **Boundary evaluation — promote:**
   - Compute `k_minus_2 = max(1, dynamic_k - 2)`
   - Collect promote candidates: tools in `scored_tools[:k_minus_2]` not in the current active set, plus tools satisfying the 2-of-3 router-proxy criterion from `state.recent_router_proxies` (Section 6)
   - `newly_promoted = session_manager.promote(session_id, promote_candidates)` — SSM is the exclusive mutation site
9. **Boundary evaluation — demote:**
   - Collect demote candidates from `state.consecutive_low_rank` per Section 5
   - `just_finished = self._just_finished_turn_used.get(session_id, set())` (CF-3)
   - `demoted = session_manager.demote(session_id, demote_candidates, just_finished, max_per_turn=3)` — SSM is the exclusive mutation site
   - Reset `state.consecutive_low_rank` for demoted tools
10. **Post-boundary state sync — in a single pass after both promote and demote:**
    - `post_boundary_active = session_manager.get_active_tools(session_id)` — authoritative from SSM
    - `state.active_tool_ids = list(post_boundary_active)` — mirrors SSM; not independently maintained
    - `state.router_enum_tool_ids = [k for k in all_registry_keys if k not in post_boundary_active]`
    - The returned direct tools for this turn are assembled from `state.active_tool_ids` (bounded to `dynamic_k`)
    - The routing tool enum for this turn uses `state.router_enum_tool_ids`
    - Newly promoted tools are in `state.active_tool_ids` and absent from `state.router_enum_tool_ids`
    - Newly demoted tools are in `state.router_enum_tool_ids` and absent from `state.active_tool_ids`
11. Write remaining scoring signals to state: `state.alpha`, `state.active_k`, `state.fallback_tier`, `state.env_confidence`, `state.conv_confidence`
12. Build `RankingEvent` with correct signal sources: `router_describes` from `_session_router_describes`, `router_proxies` from `_session_router_proxies` (CF-2); `active_tool_ids` from `state.active_tool_ids`
13. Set `_in_turn[session_id] = True`
14. Return the tool list built from `state.active_tool_ids` — **not** from `scored_tools[:dynamic_k]` directly
