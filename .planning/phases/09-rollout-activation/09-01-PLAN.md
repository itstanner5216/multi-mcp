### 4. Implement Recall@15 computation (F-11)

**File:** `src/multimcp/retrieval/replay.py`

**Source plan (line 975):** Cutover gate: "Recall@15 >=5% improvement"

Recall@15 measures: of all tool usage in a session, what fraction occurred while the used tool was in the active set? A router-proxied call means the tool was absent from the active set at the time of use and is a recall miss — unless the log contains malformed data with the tool present in both `router_proxies` and `active_tool_ids`, in which case explicit membership test takes precedence.

**Add to `ReplayMetrics` (listed in Signature / Model Changes):**
- `recall_at_15: float = 0.0`
- `canary_recall: float = 0.0`
- `control_recall: float = 0.0`

**Computation rule:** Exclude shadow-group events. Compute canary and control separately. Both groups must have at least 20 total tool-usage events for the gate to evaluate; otherwise the gate is blocked with `passed=False`.

**Add helper in `evaluate_replay()`:**
```python
def _compute_group_recall(group_events: list[dict]) -> tuple[float, int]:
    total = 0
    hits = 0
    for ev in group_events:
        active_tools = set(ev.get("active_tool_ids", []))
        for call in ev.get("direct_tool_calls", []):
            total += 1
            if call in active_tools:
                hits += 1
        for proxy_call in ev.get("router_proxies", []):
            total += 1
            if proxy_call in active_tools:
                hits += 1
    recall = hits / total if total > 0 else 0.0
    return recall, total
```

**Gate computation:**
```python
MIN_EVENTS_PER_GROUP = 20
GATE_RECALL_IMPROVEMENT = 0.05

non_shadow = [ev for ev in events if ev.get("group") != "shadow"]
canary_events = [ev for ev in non_shadow if ev.get("group") == "canary"]
control_events = [ev for ev in non_shadow if ev.get("group") == "control"]

canary_recall, canary_total = _compute_group_recall(canary_events)
control_recall, control_total = _compute_group_recall(control_events)

metrics.canary_recall = canary_recall
metrics.control_recall = control_recall
metrics.recall_at_15 = canary_recall

if canary_total < MIN_EVENTS_PER_GROUP or control_total < MIN_EVENTS_PER_GROUP:
    gates.append(CutoverGate(
        name="recall_at_15",
        passed=False,
        threshold=GATE_RECALL_IMPROVEMENT,
        actual=0.0,
        message=f"Insufficient data: canary={canary_total}, control={control_total}, need {MIN_EVENTS_PER_GROUP} per group",
    ))
else:
    recall_improvement = canary_recall - control_recall
    recall_pass = recall_improvement >= GATE_RECALL_IMPROVEMENT
    gates.append(CutoverGate(
        name="recall_at_15",
        passed=recall_pass,
        threshold=GATE_RECALL_IMPROVEMENT,
        actual=recall_improvement,
        message=f"Recall@15 improvement {recall_improvement:.1%} {'>=5%' if recall_pass else '<5%'}",
    ))
```

### 5. Fix describe_rate gate (F-11)

**File:** `src/multimcp/retrieval/replay.py` lines 149-156

**Current:** `passed=True` hardcoded — gate never blocks.

**Source plan (line 975):** "describe rate >=20% drop" — this is a blocking gate. The canary describe rate must drop by at least 20% relative to control.

**Fix:** Exclude shadow events. Compute describe rates for canary and control separately. Both groups must have at least 20 events; if not, gate is blocked with `passed=False`.

**Add to `ReplayMetrics` (listed in Signature / Model Changes):**
- `canary_describe_rate: float = 0.0`
- `control_describe_rate: float = 0.0`

**Gate computation:**
```python
MIN_EVENTS_PER_GROUP = 20
GATE_DESCRIBE_DROP = 0.20

non_shadow = [ev for ev in events if ev.get("group") != "shadow"]
canary_events = [ev for ev in non_shadow if ev.get("group") == "canary"]
control_events = [ev for ev in non_shadow if ev.get("group") == "control"]

canary_describe = _compute_describe_rate(canary_events)
control_describe = _compute_describe_rate(control_events)

metrics.canary_describe_rate = canary_describe
metrics.control_describe_rate = control_describe

canary_n = len(canary_events)
control_n = len(control_events)

if canary_n < MIN_EVENTS_PER_GROUP or control_n < MIN_EVENTS_PER_GROUP:
    gates.append(CutoverGate(
        name="describe_rate_drop",
        passed=False,
        threshold=GATE_DESCRIBE_DROP,
        actual=0.0,
        message=f"Insufficient data: canary={canary_n}, control={control_n}, need {MIN_EVENTS_PER_GROUP} per group",
    ))
else:
    if control_describe > 0:
        describe_drop = (control_describe - canary_describe) / control_describe
    else:
        describe_drop = 0.0 if canary_describe == 0 else -1.0
    describe_pass = describe_drop >= GATE_DESCRIBE_DROP
    gates.append(CutoverGate(
        name="describe_rate_drop",
        passed=describe_pass,
        threshold=GATE_DESCRIBE_DROP,
        actual=describe_drop,
        message=f"Describe rate drop {describe_drop:.1%} {'>=20%' if describe_pass else '<20%'}",
    ))
```

Where `_compute_describe_rate(events)` returns the fraction of events where `describe_tool_calls > 0`.

    if callable(rebuild):
        rebuild(registry)

**RollingMetrics.record_rescore():**
```python
def record_rescore(self) -> None:
    now = time.monotonic()
    self._rescore_times.append(now)
    # Evict old entries
    cutoff = now - self._window
    while self._rescore_times and self._rescore_times[0] < cutoff:
        self._rescore_times.popleft()

# In snapshot():
now = time.monotonic()
rescore_count_10m = sum(1 for t in self._rescore_times if t >= now - 600)
# rescore_rate_30m: all events in ring buffer / 1800s
# rescore_rate_10m: events in last 600s / 600s (alert trigger window)
snapshot.rescore_rate_30m = len(self._rescore_times) / 1800 if self._rescore_times else 0.0
snapshot.rescore_rate_10m = rescore_count_10m / 600 if rescore_count_10m else 0.0
```

**Pipeline.__init__() must receive `rolling_metrics`:** Add optional `rolling_metrics: RollingMetrics | None = None` parameter. `multi_mcp.py` creates and passes it when pipeline is constructed. This same instance is used by both `pipeline.rebuild_catalog()` (producer) and `AlertChecker` (consumer).

**Sustained-window logic for alert (source plan line 961: ">1/5s sustained for 10min"):**
`RollingMetrics` tracks rescore events with timestamps. `MetricSnapshot` exposes two rates:
- `rescore_rate_30m`: events in last 30 minutes / 1800s (general telemetry)
- `rescore_rate_10m`: events in last 10 minutes / 600s (alert trigger)

The alert fires only on `rescore_rate_10m > 0.2`. The 30-minute average alone does not trigger the alert.

### 7. Assert pipeline runtime semantics for shadow/canary/GA (F-09)

**File:** `src/multimcp/retrieval/pipeline.py`

The audit's core F-09 finding is that the rollout infrastructure is "well-tested but disconnected." Config plumbing alone (changes 1-2) does not close F-09 if the pipeline's runtime behavior for each rollout stage is not verified to work correctly.

**Mandatory implementation contract — `pipeline.py` `get_tools_for_list()` dispatch:**

| Stage | `enabled` | `shadow_mode` | `rollout_stage` | Pipeline behavior |
|-------|-----------|---------------|-----------------|-------------------|
| Off | False | * | * | Return all tools (passthrough). Short-circuit at line 67-69. |
| Shadow | True | True | shadow | Score via full BMXF+RRF path. Log RankingEvent. Return all tools. |
| Canary | True | False | canary | Control sessions: return all tools. Canary sessions: return bounded set + routing tool. |
| GA | True | False | ga | All sessions: return bounded set + routing tool. |

**Required patches to `pipeline.py`:**
- Shadow mode: `is_filtered` must be forced to `False` when `shadow_mode=True`, evaluated before the `rollout_stage` check. The shadow mode guard must be the first dispatch branch.
- Canary mode: `get_session_group()` from `rollout.py` must be called with the real `session_id` provided by Phase 8. The control branch returns all tools. The canary branch returns the bounded set + routing tool.
- GA mode: All sessions take the `is_filtered=True` branch. The bounded set + routing tool path must execute for every session.

**Mandatory dispatch order in `get_tools_for_list()`:**
```python
if not self.config.enabled:
    return list(self.tool_registry.values())  # Off: passthrough, no scoring

# ... (scoring computation runs here) ...

if self.config.shadow_mode:
    is_filtered = False                        # Shadow: score+log, no filtering
elif self.config.rollout_stage == "ga":
    is_filtered = True                         # GA: all sessions filtered
elif self.config.rollout_stage == "canary":
    is_filtered = (get_session_group(session_id, self.config.canary_percentage) == "canary")
else:                                          # "control" or unknown: passthrough
    is_filtered = False

if is_filtered:
    return bounded_tool_set + [routing_tool_def]
else:
    return list(self.tool_registry.values())
```

**Tests for each row of the table are mandatory** — see `tests/test_rollout_runtime_modes.py` (Mandatory Tests section). These replace the V-04 false claim.
